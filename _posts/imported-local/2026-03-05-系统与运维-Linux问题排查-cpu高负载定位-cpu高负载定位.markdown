uptime 命令的输出是评估系统健康状况的快速窗口，它主要包含以下四个部分：
当前时间：命令执行时的系统时间。
系统运行时长：系统自上次启动以来，已经连续运行了多长时间。
当前登录用户数：当前有多少个用户登录到了系统上。
系统平均负载：这是最关键的性能指标，反映了系统在最近 1分钟、5分钟和15分钟内的平均工作负载。
一个典型的输出示例如下：
23:30:47 up 2 days, 10:10, 1 user, load average: 0.25, 0.11, 0.07

lscpu查看cpu核数
基本原则：理想情况下，每个CPU核心处理一个进程。因此，需要计算“负载饱和度”来判断系统状态。
计算公式：负载值 / CPU逻辑核心数


Top -c 
CPU 使用率：%Cpu(s): 44.4 us, 15.9 sy, 12.6 id, 25.0 st
用户态 (us) 44.4%：用户进程消耗的CPU。
系统态 (sy) 15.9%：内核进程消耗的CPU。
空闲 (id) 12.6%：CPU只有12.6%的时间是空闲的，说明负载极重。
窃取时间 (st) 25.0%：这是最关键的指标。st 表示虚拟机被宿主机（Hypervisor）“偷走”的CPU时间。25%的窃取时间意味着这台虚拟机严重缺乏CPU资源，宿主机正在强制限制或抢占它的CPU，导致性能严重下降
