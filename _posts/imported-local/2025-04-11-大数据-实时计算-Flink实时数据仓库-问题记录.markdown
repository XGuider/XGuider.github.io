1、uid的作用
1.1 持久化状态
状态恢复：当 Flink 作业重启或从故障中恢复时，Flink 需要知道如何恢复每个算子的状态。uid 用于唯一标识每个算子，确保在作业重启后能够正确地恢复状态。
状态迁移：如果你在作业中添加或删除算子，Flink 需要知道哪些状态属于哪个算子。uid 可以帮助 Flink 在作业结构发生变化时正确地迁移状态。
1.2 调试和监控
调试：在调试 Flink 作业时，uid 可以帮助你识别和定位特定的算子，尤其是在复杂的作业中。
监控：在监控 Flink 作业时，uid 可以用于标识和跟踪特定的算子，帮助你了解算子的性能和状态。
DataStream<String> input = ...;

input
    .map(new MyMapper())
    .uid("my-mapper")
    .keyBy(/* some key */)
    .window(/* some window */)
    .reduce(new MyReducer())
    .uid("my-reducer");





2、设置 checkpoint 模式: 精准一次
Exactly-once：确保每个事件只处理一次，状态更新也只发生一次。
At-least-once：确保每个事件至少处理一次，状态更新可能发生多次。
None：不进行 Checkpoint，通常用于测试或不需要容错的场景。

StreamExecutionEnvironment env = StreamExecutionEnvironment.getExecutionEnvironment();
// 启用 Checkpoint
env.enableCheckpointing(1000); // 每 1000 毫秒进行一次 Checkpoint
// 设置 Checkpoint 模式为 Exactly-once
env.getCheckpointConfig().setCheckpointingMode(CheckpointingMode.EXACTLY_ONCE);

#// Flink 提供了多种状态后端，如 MemoryStateBackend、FsStateBackend 和 RocksDBStateBackend
env.setStateBackend(new HashMapStateBackend()); #// 设置状态后端
env.getCheckpointConfig().setCheckpointStorage("hdfs://namenode:9000/flink-checkpoints");
