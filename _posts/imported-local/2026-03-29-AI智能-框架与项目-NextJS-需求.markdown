# AI 工具聚合网站产品文档

> 一站式 AI 工具集成平台产品需求与技术架构文档

## 概述

本 AI 工具聚合网站旨在成为一个便捷的 AI 工具集成平台，将各类实用的 AI 工具整合在一起，用户能通过简单的点击操作快速访问和使用这些工具。采用 Next.js 作为前端和服务端框架，PostgreSQL 作为数据库，搭配 Clerk 进行用户认证管理，Drizzle ORM 进行数据库操作。

---

## 一、产品功能

### （一）前端功能

#### 1. 工具分类展示

以侧边栏的形式展示工具分类，如图像、音频、视频、编程等类别，方便用户快速定位。

**设计要求**：
- 分类展示需具备响应式设计
- 适应不同设备屏幕尺寸
- 支持折叠/展开

#### 2. 工具快捷使用

每个工具以卡片形式呈现，卡片上显示：
- 工具名称
- 简介
- 代表性图标

用户点击工具卡片，可直接跳转至对应 AI 工具的使用页面。

#### 3. 热门推荐

设置热门推荐区域，根据以下数据展示：
- 工具的点击量
- 用户评价

#### 4. 搜索功能

| 功能 | 说明 |
|------|------|
| 关键词搜索 | 输入工具名称、关键词 |
| 实时展示 | 搜索结果实时展示 |
| 模糊匹配 | 支持模糊匹配算法 |

#### 5. 用户个性化

| 功能 | 说明 |
|------|------|
| 工具收藏夹 | 创建常用/感兴趣工具收藏 |
| 排序设置 | 按使用频率、字母顺序等 |

#### 6. 用户反馈

设置反馈入口，用户可反馈：
- 工具使用体验
- 失效情况
- 改进建议

---

### （二）后端功能

#### 1. 工具管理系统

| 功能 | 说明 |
|------|------|
| **添加工具** | 录入名称、访问网址、简介、分类、图标等 |
| **编辑工具** | 修改简介、更换图标、修改分类 |
| **删除工具** | 删除失效或不再使用的工具 |
| **状态管理** | 设置工具的启用/禁用状态 |

#### 2. 分类管理

| 功能 | 说明 |
|------|------|
| 创建分类 | 新增分类名称和描述 |
| 编辑分类 | 修改分类信息 |
| 删除分类 | 处理分类下工具的迁移或删除 |
| 分类排序 | 调整分类展示顺序 |

#### 3. 用户认证与权限管理

- 通过 Clerk 实现注册、登录、密码重置
- 区分普通用户和管理员角色
- 分配不同的操作权限

#### 4. 数据统计与分析

| 统计项 | 说明 |
|--------|------|
| 工具点击量 | 各工具使用热度 |
| 使用次数 | 工具使用频率 |
| 用户反馈 | 反馈数量和处理情况 |
| 用户行为 | 常用工具类型、访问时段 |

#### 5. 反馈处理

- 接收用户反馈信息
- 管理员查看、标记处理状态
- 进行相应的回复或操作

## 二、技术架构

### （一）前端技术栈

| 技术 | 用途 |
|------|------|
| **Next.js** | 框架，SSR/SSG，App Router |
| **Tailwind CSS** | UI 样式，响应式设计 |
| **Headless UI** | 可访问性组件 |
| **React Context + SWR** | 状态管理和数据请求 |
| **React Hook Form + Zod** | 表单处理和验证 |
| **Framer Motion** | 动画效果 |
| **Clerk** | 用户认证 |

### （二）后端技术栈

| 技术 | 用途 |
|------|------|
| **Next.js API Routes** | API 接口 |
| **PostgreSQL** | 关系型数据库 |
| **Drizzle ORM** | 类型安全的 SQL 查询 |
| **Clerk** | 认证与授权 |
| **Vercel** | 部署平台 |

### （三）数据库设计

#### 工具表（tools）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | VARCHAR | 工具名称 |
| url | VARCHAR | 访问网址 |
| description | TEXT | 详细简介 |
| icon | VARCHAR | 图标链接 |
| category_id | UUID | 外键，分类 ID |
| is_active | BOOLEAN | 是否启用 |
| usage_count | INT | 使用次数 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

#### 分类表（categories）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | VARCHAR | 分类名称 |
| description | TEXT | 分类描述 |
| order | INT | 展示顺序 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

#### 反馈表（feedbacks）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 用户 ID（可空） |
| tool_id | UUID | 工具 ID |
| content | TEXT | 反馈内容 |
| status | VARCHAR | 处理状态 |
| created_at | TIMESTAMP | 提交时间 |
| updated_at | TIMESTAMP | 更新时间 |

#### 用户收藏表（user_favorites）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| user_id | UUID | 用户 ID |
| tool_id | UUID | 工具 ID |
| created_at | TIMESTAMP | 收藏时间 |

## 五、风险与应对

### （一）技术集成风险

**风险**：Clerk、Drizzle ORM 与 Next.js、PostgreSQL 兼容性问题

**应对**：
- 项目初期充分调研和测试
- 参考官方文档和社区案例
- 关注更新动态

### （二）数据安全风险

**风险**：用户数据泄露、篡改

**应对**：
- 采用 Clerk 安全认证机制
- 数据库加密处理
- HTTPS 传输加密
- 遵循相关法律法规

### （三）工具管理风险

**风险**：工具数量增加导致管理复杂

**应对**：
- 自动化工具状态检测
- 批量管理功能
- 定期清理失效工具
