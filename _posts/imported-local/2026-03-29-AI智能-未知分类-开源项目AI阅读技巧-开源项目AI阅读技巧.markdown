# 开源项目 AI 阅读技巧

> 使用 AI 辅助分析开源项目的核心概念、架构设计和技术选型

## 概述

本文档提供了一套使用 AI 辅助阅读和分析开源项目的提示词模板，帮助快速理解项目核心概念、模块架构、算法实现和技术选型。

## 一、核心概念体系分析

### 提示词模板

```
请提取这个项目的核心概念体系：
1. 有哪些关键术语和概念？列出完整清单
2. 每个概念的定义是什么？为什么需要这个概念？
3. 这些概念之间是什么关系？（层级/依赖/组合）
4. 哪些概念是核心的，哪些是辅助的？
5. 概念体系如何映射到代码结构？
```

### 分析维度

| 维度 | 说明 |
|------|------|
| 术语清单 | 收集项目特有的概念和术语 |
| 概念定义 | 理解每个概念的含义和作用 |
| 关系图谱 | 梳理概念间的依赖和组合关系 |
| 核心识别 | 区分核心概念和辅助概念 |
| 代码映射 | 将概念对应到具体代码模块 |

## 二、模块架构分析

### 提示词模板

```
请分析这个项目的模块架构：
1. 项目由哪些模块组成？请列出目录结构
2. 哪些是核心模块？哪些是辅助模块？
3. 每个模块的职责是什么？模块边界在哪里？
4. 模块之间的依赖关系是什么？画出依赖图
5. 每个模块的代码量是多少？（行数/文件数）
6. 代码量分布说明了什么？开发重点在哪里？
7. 哪些模块可以独立拆分？哪些强耦合？
```

### 模块分类

- **核心模块**：承担主要业务逻辑
- **辅助模块**：工具、配置、测试等支撑模块

### 分析要点

```
核心模块有哪些：承担主要业务逻辑的模块
辅助模块有哪些：工具、配置、测试等支撑模块
模块职责划分：每个模块负责什么？边界在哪里？
模块依赖关系：谁调用谁？数据如何在模块间流转？
```

## 三、核心算法分析

### 提示词模板

```
请分析这个项目的核心算法：
1. 项目使用了哪些关键算法？列出清单
2. 每个算法解决了什么问题？为什么需要它？
3. 算法的时间/空间复杂度如何？有什么限制？
4. 算法的输入输出是什么？数据结构如何？
5. 这些算法在代码中的实现在哪里？
```

### 核心问题

```
这个系统的核心能力是什么？
它能解决什么问题，不能解决什么问题？
性能瓶颈在哪里，时间/空间复杂度如何？
与其他系统的本质区别是什么？
```

## 四、技术选型评估

### 提示词模板

```
请评估这个项目的技术选型：
1. 哪些模块/算法是自研的？
2. 每个自研部分是否有成熟的替代品？列出候选方案
3. 自研的必要性是什么？是否有过度设计？
4. 如果重新设计，哪些可以替换？哪些必须保留？
5. 技术选型反映了作者的什么考虑？可能存在什么问题？
6. 从这个项目中，有哪些模块/算法值得我吸收到自己的技术储备中？
```

### 评估维度

| 问题 | 说明 |
|------|------|
| 这个模块是否必要？ | 评估模块的存在价值 |
| 自研还是集成？ | 是否有更成熟的替代品？ |
| 技术选型是否合理？ | 反映了作者的什么考虑？ |
| 如果重新设计？ | 哪些可以替换？哪些必须保留？ |

## 使用建议

1. **循序渐进**：按概念→架构→算法→选型的顺序分析
2. **结合代码**：提示 AI 时附上相关代码片段
3. **记录沉淀**：将分析结果整理成文档留存
4. **对比学习**：对比多个相似项目，发现最佳实践
