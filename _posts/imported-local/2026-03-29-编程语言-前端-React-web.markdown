# Web 开发者誓言与最佳实践

> 编程不易，因此我们要通过一切方法让它变得容易

## 概述

本文档汇总 Web 开发者的开发誓言和最佳实践，帮助建立良好的开发习惯。

---

## Web 开发者誓言

```
编程不易，因此我们要通过一切方法让它变得容易
我会始终打开我的浏览器开发者控制台
我会小步前进
我会写大量的 console.log 语句来确保我理解代码是怎么运行的，并借此准确找到问题
如果我的代码出问题了，我不会写更多的代码。而是删除代码直到它能运行，或者直接回到之前代码能运行的状态
当我在课程的 Discord 群或者其他地方寻求帮助时，我会准确表达我的问题，点此了解如何寻求帮助。
```

## 开发习惯

### 1. 善用浏览器开发者工具

| 功能 | 快捷键 | 用途 |
|------|--------|------|
| 打开控制台 | F12 / Cmd+Opt+I | 查看日志、调试 JS |
| Elements | Cmd+Shift+C | 检查 DOM 元素 |
| Network | Cmd+Opt+E | 查看网络请求 |
| Sources | Cmd+O | 查看源码文件 |

### 2. 调试技巧

```javascript
// 使用 console.log 进行调试
console.log('变量值:', variable);

// 打印对象所有属性
console.table({ key: value });

// 分组打印
console.group('请求处理');
console.log('开始处理...');
console.log('处理完成');
console.groupEnd();
```

### 3. 小步前进原则

```
遇到问题时的正确做法：
1. 写一点代码
2. 运行测试
3. 如果失败，删除或回滚
4. 重复以上步骤
```

### 4. 代码回滚策略

```bash
# Git 回滚到上一次正常工作的版本
git log
git checkout <健康的commit-id>

# 或使用 Git stash
git stash
# ... 调试
git stash pop
```

## Shell 脚本命令

### set 命令详解

| 命令 | 功能 |
|------|------|
| `set +ex` | 关闭执行跟踪（xtrace）和出错终止（errexit） |
| `set -e` | 命令失败时脚本立即停止 |
| `set +e` | 禁用 set -e，命令失败继续执行 |
| `set -a` | 将后续定义的变量标记为导出 |
| `set +a` | 取消变量自动导出 |

### exit 退出码

| 退出码 | 含义 |
|--------|------|
| `exit 0` | 成功退出 |
| `exit 1` | 失败退出（通用错误） |

### echo 命令

```bash
# 使用 -e 选项解释转义字符
echo -e "第一行\n第二行"
echo -e "字段1\t字段2"
```

### top 命令

```bash
# 每5秒刷新一次数据，共刷新10次后退出
top -d 5 -n 10
```

## Vi/Vim 编辑器快捷键

### 删除

| 操作 | 按键 |
|------|------|
| 删除光标所在字符 | `x` |
| 删除光标所在行 | `dd` |
| 删除光标到行尾 | `D` |

|------|------|

| 操作 | 按键 |
|------|------|
