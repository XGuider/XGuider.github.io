# ClkLog 项目架构分析

> 数据采集平台完整架构

🔗 [在线演示](https://demo.clklog.com/#/userBehavior/userLoyalty)

---

## 🏗️ 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      数据采集平台架构                          │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐│
│  │ 数据上送  │ -> │ 数据处理  │ -> │ 数据分析  │ -> │ 平台管理  ││
│  │  Service │    │  Service │    │   API    │    │  接口    ││
│  └──────────┘    └──────────┘    └──────────┘    └──────────┘│
└─────────────────────────────────────────────────────────────┘
```

---

## 1️⃣ 数据上送

### 1.1 SDK 构建（神策 SDK）

**处理逻辑：**
1. 接受消息后，封装成对象
2. 解析消息添加 UA 和 IP 等信息
3. 推送消息队列
4. 初始化任务，加载映射关系到内存中
5. 线程池线程定时消费消息队列消息，推送给接受服务

### 1.2 数据接受服务

| 方式 | 处理逻辑 |
|------|----------|
| **Kafka** | 拼接字符串，直接封装推送 |
| **ClickHouse** | 解析出地址位置（IP映射的位置缓存在Redis中），批量写入CK |

---

## 2️⃣ 数据处理服务（Flink）

1. 创建流环境、Checkpoint
2. 构建 Kafka Source，映射处理正确的数值，string2object
3. Open 初始化该初始化的数据
4. 窗口，批量下发数据，写入 CK Sink

---

## 3️⃣ 数据分析服务（API）

### 分析维度

| 维度 | 说明 |
|------|------|
| **人** | 用户属性相关 |
| **货** | 页面相关 |
| **场** | 系统、地域、渠道 |

### 技术实现
- `@PostConstruct` 对象创建后执行初始化

---

## 4️⃣ 数据定时任务

### 统计表

| 表名 | 说明 |
|------|------|
| 用户详细表 | 用户基本信息 |
| 用户详细信息访问表 | 用户访问记录 |
| 用户行为分析统计表 | 行为统计 |
| 地域详情表 | 地域分布 |
| 受访页面详情表 | 页面访问 |
| 用户访问频次分布表 | 访问频次 |
| 新老访客生命周期详情表 | 用户生命周期 |
| 设备详情表 | 设备信息 |
| 受访页面表 | 页面统计 |

---

## 5️⃣ 平台管理接口

### 5.1 用户管理

- 登陆/登出接口
- 通过 AOP 进行用户操作记录
- 通过 Security 进行鉴权
- 通过 Spring 进行数据库查询

### 5.2 项目管理

- 全局设置
- 定时任务加载配置进入 Redis

---

## 🔧 技术栈

- **后端**: Spring Boot
- **消息队列**: Kafka
- **数据库**: ClickHouse, Redis, MySQL
- **流处理**: Flink
- **安全**: Spring Security
