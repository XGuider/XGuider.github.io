1. 使用场景
- Dockerfile：构建单个应用的镜像
- docker-compose：编排多个容器的运行环境
2. 配置范围
- Dockerfile：关注容器内部环境
- docker-compose：关注容器间的关系和整体应用架构


# 构建镜像
docker build -t quick-buy-ai .
# 运行容器
docker run quick-buy-ai


# 启动所有服务
docker-compose up
# 停止所有服务
docker-compose down
