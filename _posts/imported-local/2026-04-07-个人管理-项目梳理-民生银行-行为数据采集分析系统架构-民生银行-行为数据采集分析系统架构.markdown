行为数据是标签、圈客、画像不可缺少的数据，行为数据采集涉及行内各端IOS、ANDROID以及各个端数据采集，涉及3亿的PV，存储接近PB的量级；

- 金融级数据中台 | 信创迁移(MySQL) | 实时数仓(Hudi) | K8s故障诊断 | 数据血缘治理 | 智能问答(NLP)

# 角色：项目管理

# 主要工作：
	- 全链路系统迁移与架构升级
```
主导PB级行为数据采集分析系统的新老架构迁移，完成日均3亿PV的数据管道无缝切换
设计并实施多环境兼容方案（IPv6/Nginx/Apache），系统吞吐量提升40%
建立标准化上线流程，涵盖资源申请、上下游服务验证、灰度发布及监控体系
```
	- 信创改造与灾备体系建设
```
牵头MySQL信创替代方案，设计双活数据同步策略，保障金融级数据一致性
构建跨机房灾备系统，实现RPO<15s的关键业务数据保护
攻克ID主体加密转换技术难题，研发加解密中间件降低业务侵入性
```
	- 新系统支持以及二次开发（血缘、智能问答）；负责新系统的问题排查，以及现有的二次开发
```
开发数据血缘追踪系统，实现字段级血缘关系可视化，问题定位效率提升60%
搭建智能问答引擎，支持自然语言查询数据质量指标（响应时间<200ms）
主导Hudi实时数仓适配，解决CDC数据与离线批处理系统的时序一致性问题
```
	- 云原生技术落地
```
作为K8s技术专家解决容器化部署难题，优化JVM参数使GC时间降低70%
设计品牌数据异常检测算法，准确识别98%以上的脏数据
```

# 主要成果

	系统稳定性：实现全年99.99% SLA，故障MTTR<30分钟
	性能指标：Nginx改造后QPS提升至50k，资源消耗降低35%
	业务价值：支撑精准营销系统上线，标签计算时效性从T+1提升至分钟级



# 困难 

- 技术深度问题准备：
	"如何设计金融级数据加密迁移方案？" → 可结合ID转换案例讲解双因素认证+国密算法实践
	"Hudi在实时数仓中的取舍" → 对比Kafka Connect与Hudi的CDC处理差异


- 架构设计问题准备：
	绘制系统演进图：从单体采集→微服务化→云原生化的转型路径
	重点说明技术决策依据（如选型Nginx而非Apache的流量整形需求）
