### 1. Hive的内部表和外部表有什么区别？在什么场景下会使用外部表？

**核心区别**

- **内部管理表（Internal/Managed Table）**：Hive“全权负责”。创建表时，Hive将数据移动到其管理的仓库目录中；**删除表时，元数据和HDFS上的实际数据都会被删除**。
- **外部表（External Table）**：Hive“只读不写”。创建表时，Hive仅记录数据路径（Location）；**删除表时，只删除元数据，HDFS上的数据保留不动**。

### 2.请解释一下Hive的几种排序方式
|   |   |   |   |
|---|---|---|---|
|**ORDER BY**|Reduce端|✅ 是|对所有数据进行**全局排序**。只有一个Reduce任务处理，数据量大时效率极低，容易OOM。|
|**SORT BY**|Reduce端|❌ 否|在每个Reduce内部进行**局部排序**。不保证整体有序，通常配合`DISTRIBUTE BY`使用。|
|**DISTRIBUTE BY**|Map端|❌ 否|类似MR的Partition，控制数据**如何分发**到Reduce（如按user_id分发），保证相同key的数据进入同一个Reduce。|
|**CLUSTER BY**|Map+Reduce|❌ 否|是`DISTRIBUTE BY` + `SORT BY`的简写。既按字段分发，又按该字段排序。|


### 3. Hive SQL中ROW_NUMBER(), RANK(), DENSE_RANK()的区别

这三个都是窗口函数，用于处理排名，区别在于**如何处理并列（重复）名次**：

- **ROW_NUMBER()**：**不重复，不并列**。
    - 即使数值相同，也会强制分配不同的序号（1, 2, 3, 4...）。适用于“取Top N”场景（如取每个班级成绩最高的前3名学生）。
- **RANK()**：**跳跃排序**。
    - 如果有并列，名次相同，但会占用后续名次（1, 2, 2, 4...）。
- **DENSE_RANK()**：**连续排序**。
    - 如果有并列，名次相同，但不占用后续名次（1, 2, 2, 3...）


### 4. 如何处理Hive中的数据倾斜问题？（高频考点）

数据倾斜是指Map或Reduce任务中，某些任务处理的数据量远超其他任务，导致整体运行缓慢。

**常见场景与解决方案**

1. **Group By 导致的倾斜**
    
    - **现象**：少数几个Key的数据量极大（如“未知”或“测试账号”）。
    - **解决**：开启Map端聚合。
        - 设置 `hive.map.aggr=true`（在Map端先进行一次预聚合，减少传输到Reduce的数据量）。
        - 设置 `hive.groupby.skewindata=true`（生成两个MR任务，第一个随机分发进行聚合，第二个按Key聚合，实现负载均衡）。
2. **Join 导致的倾斜**
    
    - **大表 Join 小表**：
        - **解决**：使用**Map Join**。将小表加载到内存广播到所有Map节点，在Map端完成Join，避开Reduce阶段。
    - **大表 Join 大表（且存在空值或热点Key）**：
        - **解决（空值）**：过滤掉关联字段的空值，或者给空值赋随机值，将其分散到不同Reduce。
        - **解决（热点Key）**：**加盐（Salting）**。给大表的热点Key加上随机前缀（0-9），将小表膨胀（复制10份对应前缀），进行关联后再去前缀。
3. **采样与定位**

### 5. 什么是缓慢变化维（SCD）？你处理过哪几种？

SCD（Slowly Changing Dimension）指维度表中属性的值会随时间缓慢变化。

- **SCD Type 1（覆盖）**：直接更新旧数据，不保留历史。适用于修正错误或不重要的属性变更。
- **SCD Type 2（拉链表）**：**最常用**。保留历史状态。
    - **实现**：表中增加`start_date`（生效时间）和`end_date`（失效时间）。当属性变更时，将旧记录的`end_date`改为变更时间，并插入一条新记录。
    - **应用**：例如用户的会员等级变化、归属地变化，需要回溯历史数据时使用。
- **SCD Type 3（增加列）**：在表中增加“当前值”和“原值”列，只保留最近一次的历史
